# MD 파일 사용법

이 문서는 프로젝트 안의 `.md` 파일을 언제, 어떻게 참고할지 정리한 안내입니다.
토큰 절약을 위해 필요한 문서만 읽습니다.

## 한눈에 보기

| 파일 | 용도 | 언제 읽나 |
| --- | --- | --- |
| `AGENTS.md` | Codex/Claude 공통 최상위 작업 기준 | 모든 작업 시작 전 |
| `CLAUDE.md` | Claude 전용 진입 안내 | Claude로 작업할 때 |
| `docs/ui-rules.md` | UI, 타이포그래피, 버튼 크기 세부 기준 | UI/CSS/반응형 작업할 때 |
| `docs/md-usage.md` | MD 파일 사용 안내 | 어떤 문서를 볼지 헷갈릴 때 |

## 기본 사용 순서

1. 먼저 `AGENTS.md`를 확인합니다.
2. Claude로 작업하면 `CLAUDE.md`도 확인합니다.
3. UI 세부 기준이 필요할 때만 `docs/ui-rules.md`를 확인합니다.
4. 문서끼리 내용이 충돌하면 `AGENTS.md`를 우선합니다.

## 작업별 참고 문서

### 일반 수정

- 필수: `AGENTS.md`
- 선택: 없음

기존 화면, 기능, 반응형을 유지하면서 요청받은 파일만 최소 수정합니다.

### Claude와 Codex를 이어서 작업

- 필수: `AGENTS.md`
- Claude 사용 시: `CLAUDE.md`

이전 AI가 만든 구조를 먼저 읽고, 새로 만들기보다 기존 작업을 이어서 수정합니다.

### UI / CSS / 반응형 수정

- 필수: `AGENTS.md`
- 필요 시: `docs/ui-rules.md`

타이포그래피, 버튼 크기, 섹션 구조, 반응형 기준이 필요할 때만 UI 문서를 확인합니다.

### 문서 수정

- 필수: `AGENTS.md`
- 필요 시: `docs/md-usage.md`

새 문서를 만들기보다 기존 문서를 필요한 만큼만 보강합니다.
중복 내용은 만들지 않습니다.

## 답변 기록 형식

작업 후에는 길게 설명하지 않고 아래 형식으로 짧게 남깁니다.

```text
수정:
유지:
확인:
참고:
```

## 주의

- 모든 `.md` 파일을 매번 다 읽지 않습니다.
- 작업과 직접 관련 없는 문서는 열지 않습니다.
- 같은 규칙을 여러 문서에 반복해서 쓰지 않습니다.
- 문서가 길어지면 핵심만 남기고 줄입니다.
